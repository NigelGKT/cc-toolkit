# Playbooks — Cross-Project Lessons

Generalizable, **client-agnostic** knowledge that isn't a command or a skill:
consulting patterns, gotchas, and checklists distilled from real project work.

## What belongs here
- Reusable lessons that apply across clients/projects
  (e.g. "PNL data with timezone drift → checklist to diagnose").
- Patterns worth repeating on the next engagement.

## What does NOT belong here
- Client names, credentials, proprietary data, or project-specific detail —
  that stays in the project's own repo + wiki.
- Anything that is really a command or a skill — file those under
  `commands/` or `skills/` instead.

## How it fills up
The `s.wrap-up` ritual flags promote candidates at the end of each project
session (Part C). Client-agnostic lessons land here as short notes.
Optionally run `s.wiki` against this folder so it compounds over time.

One lesson per file, kebab-case filename. Keep entries short and scrubbed.
